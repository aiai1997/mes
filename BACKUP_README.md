# 数据备份和恢复功能

## 功能概述

MES系统的数据备份和恢复功能提供了完整的企业级数据保护解决方案，支持：

- **自动备份**：定时创建完整数据备份
- **手动备份**：管理员可随时创建备份
- **数据恢复**：从备份文件恢复系统数据
- **备份管理**：查看、下载、删除备份文件
- **操作日志**：记录所有备份和恢复操作

## 技术实现

### 数据库备份
- 使用 PostgreSQL 的 `pg_dump` 工具创建完整备份
- 支持 `--no-owner --no-privileges` 参数，确保备份文件可在不同环境恢复
- 备份文件以 SQL 格式存储，便于查看和手动恢复

### 权限控制
- 备份创建：`backup:create` 权限
- 备份恢复：`backup:restore` 权限
- 备份删除：`backup:delete` 权限
- 菜单访问：`menu:backup` 权限

### 文件管理
- 备份文件存储在 `backups/` 目录
- 文件命名格式：`mes_backup_{type}_{timestamp}.sql`
- 支持文件大小显示和创建时间记录

## API 接口

### 获取备份列表
```
GET /api/backup
```

### 创建备份
```
POST /api/backup
Content-Type: application/json

{
  "backupType": "full"
}
```

### 恢复备份
```
PUT /api/backup?action=restore
Content-Type: application/json

{
  "backupName": "mes_backup_full_2024-01-01T10-00-00-000Z"
}
```

### 删除备份
```
DELETE /api/backup
Content-Type: application/json

{
  "backupName": "mes_backup_full_2024-01-01T10-00-00-000Z"
}
```

### 下载备份
```
GET /api/backup/download/{backupName}
```

## 使用说明

### 创建备份
1. 进入"数据备份"页面
2. 点击"创建备份"按钮
3. 系统将自动创建完整的数据备份
4. 备份完成后会显示在列表中

### 恢复数据
1. 在备份列表中选择要恢复的备份文件
2. 点击"恢复"按钮
3. 确认恢复操作（此操作不可撤销）
4. 系统将从备份文件恢复数据
5. 恢复完成后页面会自动刷新

### 下载备份
1. 在备份列表中点击"下载"按钮
2. 备份文件将下载到本地
3. 可用于手动恢复或归档保存

### 删除备份
1. 在备份列表中点击"删除"按钮
2. 确认删除操作
3. 备份文件将被永久删除

## 安全注意事项

- **恢复操作**：恢复数据将覆盖当前所有数据，请谨慎操作
- **备份验证**：建议定期验证备份文件的完整性
- **权限控制**：只有系统管理员可以进行备份和恢复操作
- **操作日志**：所有操作都会记录在操作日志中，便于审计

## 故障排除

### 备份失败
- 检查数据库连接配置
- 确认 PostgreSQL 客户端工具已安装
- 查看系统日志获取详细错误信息

### 恢复失败
- 确认备份文件完整性
- 检查数据库权限
- 确认目标数据库为空或可覆盖

### 权限问题
- 确认用户具有相应权限
- 检查 RBAC 配置是否正确初始化
